1. Schemas
1) auth - Хранится все пользователи, и инфа о них
2) app - Где хранится, основаня сущности приложения
3) ref - Справочники


2. Tables
2.1 auth
user - данные для регистрации, входа, выхода
profile - мягкие данные о пользователе
session - сессия пользователя
token - токены, для запросов

2.2 app
generated_documents
user_id, document_id, document_text

2.3 - ref
documents - документы, которые можно сгенерить
stages - стадия, на которой можно сгенерить док
